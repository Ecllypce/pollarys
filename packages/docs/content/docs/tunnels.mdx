# Tunnels

Use `pollarys tunnel` to expose a running Pollarys instance.

## Quick start (Cloudflare quick mode)

1. Start Pollarys:

```bash
pollarys
```

2. Start a tunnel:

```bash
pollarys tunnel start --provider cloudflare --mode quick
```

3. Check status:

```bash
pollarys tunnel status
```

By default, Pollarys prints a QR code in interactive TTY sessions. Use `--qr` to force QR output, or `--no-qr` to disable it.

## Managed modes

### Managed remote

Use a token + hostname managed by Cloudflare:

```bash
pollarys tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Managed local

Use a local `cloudflared` config:

```bash
pollarys tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Profiles (managed-remote)

Save a reusable profile:

```bash
pollarys tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Start using the saved profile:

```bash
pollarys tunnel start --profile prod-main
```

## Useful commands

```bash
pollarys tunnel providers
pollarys tunnel ready --provider cloudflare
pollarys tunnel doctor --provider cloudflare
pollarys tunnel stop --port 3000
```

## Behavior notes

- one active tunnel per Pollarys instance (port)
- starting a new mode/provider on same instance replaces previous tunnel
- generating a new connect link revokes previous unused one
